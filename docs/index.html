Código de exemplo do livro sobre Claude Code

Crie e edite Google Forms conversando com o Claude

mcp-server-google-forms é um servidor MCP local, em Node.js, com 15 ferramentas para o Claude criar formulários completos numa única operação, montar quizzes com gabarito, organizar seções, publicar e ler respostas — tudo pela conversa.

  • MCP Registry oficial (da Anthropic e da comunidade MCP) — registrado como io.github.claude-book/mcp-server-google-forms.
  • Zenodo (CERN) — cada versão é arquivada com DOI permanente; citável em trabalhos acadêmicos.

Como funciona

O servidor roda no seu computador e conversa com a API oficial do Google Forms usando uma autorização que só você concede — com os escopos mínimos (formulários e leitura de respostas; nada de acesso ao Drive inteiro). Nenhum dado passa por servidores de terceiros: é a sua máquina falando diretamente com o Google.

O que ele faz (15 ferramentas)

  • Criar — build_form monta o formulário inteiro numa única operação (título, descrição, quiz e até 50 perguntas); create_form cria um vazio.
  • Perguntar — add_question com 9 tipos: texto curto e longo, escolha única, caixas de seleção, lista suspensa, escala linear, data, hora/duração e avaliação (estrelas, corações, joinhas).
  • Organizar — seções (add_section), blocos de texto (add_text_item), mover (move_question) e apagar (delete_question) com validação de posição.
  • Editar sem recriar — update_question e update_form_info preservam o vínculo com respostas já recebidas.
  • Quiz de verdade — pontos e gabarito por pergunta (set_quiz), com auditoria pós-criação (verify_answer_keys).
  • Publicar e acompanhar — set_publish, get_form (resumo legível) e list_responses (paginada).
  • Diagnóstico — auth_status testa as credenciais direto com o Google.

E quando algo dá errado, as mensagens explicam o que fazer, em português — nada de erro técnico cru.

Manual: comece a usar

Nunca mexeu com terminal? Sem problema. O passo a passo abaixo foi escrito para quem não programa: é só copiar e colar os comandos. Você faz esta configuração uma única vez; depois, é só conversar com o Claude.

Antes de começar (o que você precisa ter)

  • Node.js 18 ou mais novo — é o programa que faz o servidor rodar; os comandos npm e npx vêm junto. Para conferir, abra o terminal e digite node --version. Se não tiver, baixe a versão LTS em nodejs.org.
  • O Claude Code instalado — é por ele que você conversa com o servidor (code.claude.com/docs).
  • Uma conta Google, a mesma em que os formulários vão aparecer.

Como abrir o terminal: no Windows, menu Iniciar → digite PowerShell → abra o Windows PowerShell. No Mac, Cmd + Espaço → Terminal. No Linux, procure por Terminal (ou Ctrl + Alt + T).

Passo 1 — Preparar o Google Cloud (uma vez)

Isto autoriza o servidor a falar com o Google Forms em seu nome. Acesse console.cloud.google.com e faça login.

  1. Crie um projeto. No topo, clique no seletor de projetos → Novo projeto → dê um nome → Criar.
  2. Ative a Google Forms API. Na busca do topo, procure por Google Forms API, abra e clique em Ativar.
  3. Configure a tela de permissão. Menu → APIs e serviços → Tela de permissão OAuth → tipo Externo → preencha o nome do app e os e-mails de contato (podem ser os seus).
  4. Adicione-se como usuário de teste — o passo que mais gente esquece. Na Tela de permissão, em Usuários de teste, clique em Adicionar usuários e coloque o seu próprio e-mail do Google. Sem isso, o Google recusa a autorização com “acesso negado”.
  5. Crie a credencial. Menu → APIs e serviços → Credenciais → Criar credenciais → ID do cliente OAuth → tipo App para computador → Criar.
  6. Baixe o arquivo. Clique em Fazer download do JSON. Esse é o seu client_secret*.json.

Passo 2 — Instalar e autorizar

Escolha um dos dois caminhos. O Caminho A é o mais simples e não exige baixar o código.

Caminho A — via npm (recomendado, sem clonar nada)

  1. Rode o comando de autorização uma primeira vez. Ele vai falhar de propósito, mas já cria a pasta certa e mostra o caminho dela: npx -p mcp-server-google-forms mcp-server-google-forms-token. Vai aparecer algo como “Não encontrei nenhum 'client_secret*.json' em …/.config/mcp-server-google-forms”. Copie esse caminho.
  2. Coloque o client_secret*.json naquela pasta. Ela é oculta, então use o atalho de “ir para a pasta”: no Windows, no Explorer, cole na barra de endereço %USERPROFILE%\.config\mcp-server-google-forms e dê Enter. No Mac, no Finder, Cmd + Shift + G e cole ~/.config/mcp-server-google-forms. No Linux, Ctrl + L e cole o mesmo caminho.
  3. Rode o mesmo comando de novo — agora o navegador abre. Escolha sua conta; na tela “O Google não verificou este app” (normal, pois o app é seu), clique em Avançado → Acessar … (não seguro) e confirme as permissões. No terminal aparece “Pronto. refreshToken salvo…”.
  4. Registre no Claude Code: claude mcp add google-forms -- npx mcp-server-google-forms

Por que a pasta fica “oculta”? O caminho começa com um ponto (.config) e, por isso, a pasta não aparece por padrão no Explorer (Windows) nem no Finder (Mac). Isso é intencional.

Programas costumam guardar suas configurações — preferências, credenciais, chaves de acesso — em uma pasta reservada, para não espalhar arquivos pela sua pasta de usuário. Por convenção, essas pastas ficam agrupadas em ~/.config (o ~ representa a sua pasta de usuário), e o ponto no início do nome faz o sistema não exibi-las na navegação normal. No Windows, o caminho equivalente é C:\Users\SeuNome\.config\mcp-server-google-forms, e essa pasta também fica oculta por padrão.

“Oculta” não significa “protegida”: a pasta é sua e pode ser aberta a qualquer momento — foi o que fizemos ao colar o caminho na barra de endereço. O ponto apenas evita que ela apareça junto dos seus documentos. É o local padrão onde ferramentas de linha de comando esperam encontrar esse tipo de arquivo, e por isso o servidor guarda ali o seu client_secret.json e a autorização.

Caminho B — via clone do repositório (para quem for mexer no código)

  1. Clone o repositório e rode npm install.
  2. Coloque o client_secret*.json na pasta credentials/ dentro do projeto (crie-a se não existir).
  3. Rode npm run token e autorize no navegador (mesma tela do Google descrita no Caminho A, incluindo o aviso de “app não verificado”).
  4. Registre no Claude Code: claude mcp add google-forms -- node "$(pwd)/src/server.js"

Passo 3 — Peça ao Claude

Pronto: peça ao Claude, por exemplo, “crie um quiz de 5 perguntas sobre fotossíntese, valendo 2 pontos cada, e me dê o link”. O passo a passo completo, com solução de problemas, está no README.
